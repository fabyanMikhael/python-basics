Variables üì¶

To do most things in code we must be able to save/hold onto data so that we can use it later
With python we can do this by:

variable name = value
in code: 
dozen = 12
This creates a variable with the name of dozen that has the value of 12
in the background this looks something like:


The variable Dozen points to an object which represents the value 12, keep this in mind as it helps to clear up some confusing behavior that you will see later (hopefully idk i haven't thought this over)

another example

                x = 10
y = 3.65
z = "hello"
this will create three variables: x, y, and z
x will have the value of 10 (points to a int object with value of 10)
y will have the value of 3.65 (points to a float object with value of 3.65)
z will have the value of the string "hello" (points to a string object with value of hello)

variables can always be re-assigned
Python does not allow you to make a variable immutable (Objects however can have immutable properties): 
            x = 52
x = 1.0
x = "can be changed to any type"
There is no particular syntax to tell when a variable is declared, the first assignment is just where the variable is created
The variable x here gets declared and assigned the value 52, which is then changed to 1.0 and finally re-assigned to a string
Python is dynamically typed and variables do not have to stick to a certain type throughout the code execution

Python's Primitive Types üîç

Each Object in python has a type which determines what it is and gives you a hint at what it represents
These types are defined by classes- which are definitions that specify how an object should be built, what data it has and what it can do
one thing I should note that is I have used a new syntax for declaring variables,
this syntax allows you to explicitly set the type of the variable
here's the format: 
[variable name]: [type] = [value]


note however that this does not enforce the variable to hold only that type, it is just for reading and autocompletion purposes
The basic types in python are:

int (Integer) üî¢

represents a whole number.
In newer python versions this type has no limit, you can set it to a number that is as high as you want
(-) negative and (+) positive 
x: int = 5

float (Float-point number) #Ô∏è‚É£

represents a decimal number
(-) negative and (+) positive 
x: float = 2.7

str (String) üè∑Ô∏è

represents a piece of text (a collection of 0, 1 or more characters) 
x: str = "this is a string"
single quotes also work 
x: str = 'hello there!'

bool (Boolean) üîÑ

can be only be either True or False 
x: bool = False

x: None = None

Python's Collection Types üßÆ

list üìó

A collection type that can hold many objects (useful for holding hundreds or thousands of objects instead of making a variable for each)
You can make one with the [a,b,c,...] syntax 
x: list = [1,10,23]
You can also have any type you want in it 
x: list = [4.325, 7, "hello"]
Retrieving an element from the list is done with the index syntax: array[index] 
x: list = [4.325, 7, "hello"] 
num : float = x[0]
num2: int   = x[1]
hi  : str   = x[2]
print(x)    #prints 4.325
print(num2) #prints 7
print(hi)   #prints hello
note that the indices for lists start at 0, so the first element is actually at the 0th index not 1
ALSO the types (the : float part of x: float) I am writing are unneccessary but you should write them to get used to it (also helps with code completion)

Dictionary üìí

maps primitive or hashable objects to values of any type
you can create a dictionary with the {key: value,...} constructor 
dictionary: dict = {'key': 'value', 3: 'three'}
This now means that in the dictionary:
input --> output
'key' --> 'value'
3 --> 'three'

retrieving an element from the dictionary is done by using an existing key with this syntax: dictionary[key] 
dictionary: dict = {'key': 'value', 3: 'three'}
key_item = dictionary['key'] #will be the string 'value'
three = dictionary[3] #will be the string 'three'
adding and setting a value of a key is done in the same way: dictionary[key] = new_value 
dictionary = {}
dictionary['new val'] = 145.25
dictionary['new val'] = 'changing the value'
finally, deleting a key/value pair can be done with: del dictionary[key] 
dictionary = {}
dictionary['delete me'] = ["you can nest lists", "and", "dictionaries"]
del dictionary['delete me'] # 'delete me' is now gone
print(dictionary['delete me']) # this will cause an error
